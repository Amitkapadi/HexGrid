Detailed Description

Represents the position of a hex within a hex grid using axial coordinates (q = East, r = Southeast). This results in a grid with a naturally parallelgraphic form, but can still be stored in a regtangular array resulting in a small number of wasted cells.

This type is the less computationally efficient type to use for hex grid computations than CubicHexCoord, as all of the work has to be done by the CubicHexCoord type and converting between AxialHexCoord and CubicHexCoord is a small cost every time a result must be obtained.
